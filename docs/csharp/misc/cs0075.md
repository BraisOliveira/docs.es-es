 <span data-ttu-id="2bdd3-107">*Desde la regla de desambiguación se deduce que, si x e y son identificadores, (x) y, (x)(y), y (son son expresiones de conversión, pero (x)-y es no, aunque x identifique un tipo. Sin embargo, si x es una palabra clave que identifica un tipo predefinido (por ejemplo, int), a continuación, las cuatro formas son expresiones de conversión (porque esta palabra clave no podría ser una expresión por sí mismo).*</span><span class="sxs-lookup"><span data-stu-id="2bdd3-107">*From the disambiguation rule it follows that, if x and y are identifiers, (x)y, (x)(y), and (x)(-y) are cast-expressions, but (x)-y is not, even if x identifies a type. However, if x is a keyword that identifies a predefined type (such as int), then all four forms are cast-expressions (because such a keyword could not possibly be an expression by itself).*</span></span>  
  
 <span data-ttu-id="2bdd3-108">El código siguiente genera el error CS0075:</span><span class="sxs-lookup"><span data-stu-id="2bdd3-108">The following code generates CS0075:</span></span>  
  
```csharp  
// CS0075  
namespace MyNamespace  
{  
    enum MyEnum { }  
    public class MyClass  
    {  
        public static void Main()  
        {  
            // To fix the error, place the negative  
            // values below in parentheses  
            int i = (System.Int32) - 4; //CS0075  
            MyEnum e = (MyEnum) - 1;    //CS0075  
            System.Console.WriteLine(i); //to avoid warning  
            System.Console.WriteLine(e); //to avoid warning  
        }  
    }  
}  
```
