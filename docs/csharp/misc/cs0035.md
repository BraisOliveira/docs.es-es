# <a name="compiler-error-cs0035"></a><span data-ttu-id="9aa67-102">Error del compilador CS0035</span><span class="sxs-lookup"><span data-stu-id="9aa67-102">Compiler Error CS0035</span></span>
<span data-ttu-id="9aa67-103">El operador 'operator' es ambiguo en un operando del tipo 'type'</span><span class="sxs-lookup"><span data-stu-id="9aa67-103">Operator 'operator' is ambiguous on an operand of type 'type'</span></span>  
  
<span data-ttu-id="9aa67-104">El compilador tiene más de una conversión disponible y no sabe cuál debe realizar antes de aplicar el operador.</span><span class="sxs-lookup"><span data-stu-id="9aa67-104">The compiler has more than one available conversion and does not know which to choose before applying the operator.</span></span>
  
<span data-ttu-id="9aa67-105">El ejemplo siguiente genera la advertencia CS0035:</span><span class="sxs-lookup"><span data-stu-id="9aa67-105">The following sample generates CS0035:</span></span>  
  
```csharp  
// CS0035.cs  
class MyClass  
{  
   private int i;  
  
   public MyClass(int i)  
   {  
      this.i = i;  
   }  
  
   public static implicit operator double(MyClass x)  
   {  
      return (double) x.i;  
   }  
  
   public static implicit operator decimal(MyClass x)  
   {  
      return (decimal) x.i;  
   }  
}  
  
class MyClass2  
{  
   static void Main()  
   {  
      MyClass x = new MyClass(7);  
      object o = - x;   // CS0035  
      // try a cast:  
      // object o = - (double)x;  
   }  
}  
```
