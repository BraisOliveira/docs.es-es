# <a name="nullable-value-types-c-programming-guide"></a><span data-ttu-id="b1726-103">Tipos de valor que admiten un valor NULL (Guía de programación de C#)</span><span class="sxs-lookup"><span data-stu-id="b1726-103">Nullable value types (C# Programming Guide)</span></span>

<span data-ttu-id="b1726-104">Los tipos de valor que admiten un valor NULL son instancias de la estructura <xref:System.Nullable%601?displayProperty=nameWithType>.</span><span class="sxs-lookup"><span data-stu-id="b1726-104">Nullable value types are instances of the <xref:System.Nullable%601?displayProperty=nameWithType> structure.</span></span> <span data-ttu-id="b1726-105">Los tipos de valor que admiten un valor NULL pueden representar todos los valores de un tipo `T` subyacente y un valor [NULL](../../language-reference/keywords/null.md) adicional.</span><span class="sxs-lookup"><span data-stu-id="b1726-105">Nullable value types can represent all the values of an underlying type `T`, and an additional [null](../../language-reference/keywords/null.md) value.</span></span> <span data-ttu-id="b1726-106">EL tipo `T` subyacente puede ser cualquier [tipo de valor](../../language-reference/keywords/value-types.md) que no acepte valores NULL.</span><span class="sxs-lookup"><span data-stu-id="b1726-106">The underlying type `T` can be any non-nullable [value type](../../language-reference/keywords/value-types.md).</span></span> <span data-ttu-id="b1726-107">`T` no puede ser un tipo de referencia.</span><span class="sxs-lookup"><span data-stu-id="b1726-107">`T` cannot be a reference type.</span></span>

> [!NOTE]
> <span data-ttu-id="b1726-108">C# 8.0 presenta la característica de tipos de referencia que admiten valores NULL.</span><span class="sxs-lookup"><span data-stu-id="b1726-108">C# 8.0 introduces the nullable reference types feature.</span></span> <span data-ttu-id="b1726-109">Para más información, vea [Tipos de referencia que aceptan valores NULL](../../nullable-references.md).</span><span class="sxs-lookup"><span data-stu-id="b1726-109">For more information, see [Nullable reference types](../../nullable-references.md).</span></span> <span data-ttu-id="b1726-110">Los tipos de valor que admiten valores NULL están disponibles a partir de C# 2.</span><span class="sxs-lookup"><span data-stu-id="b1726-110">The nullable value types are available starting with C# 2.</span></span>

<span data-ttu-id="b1726-111">Por ejemplo, puede asignar `null` o cualquier valor entero de <xref:System.Int32.MinValue?displayProperty=nameWithType> a <xref:System.Int32.MaxValue?displayProperty=nameWithType> a un elemento `Nullable<int>` y [true](../../language-reference/keywords/true-literal.md), [false](../../language-reference/keywords/false-literal.md) o `null` a un elemento `Nullable<bool>`.</span><span class="sxs-lookup"><span data-stu-id="b1726-111">For example, you can assign `null` or any integer value from <xref:System.Int32.MinValue?displayProperty=nameWithType> to <xref:System.Int32.MaxValue?displayProperty=nameWithType> to a `Nullable<int>` and [true](../../language-reference/keywords/true-literal.md), [false](../../language-reference/keywords/false-literal.md), or `null` to a `Nullable<bool>`.</span></span>

<span data-ttu-id="b1726-112">Los tipos de valor que admiten valores NULL se usan cuando es necesario representar el valor indefinido de un tipo subyacente.</span><span class="sxs-lookup"><span data-stu-id="b1726-112">You use a nullable value type when you need to represent the undefined value of an underlying type.</span></span> <span data-ttu-id="b1726-113">Una variable booleana solo puede tener dos valores: `true` y `false`.</span><span class="sxs-lookup"><span data-stu-id="b1726-113">A Boolean variable can have only two values: `true` and `false`.</span></span> <span data-ttu-id="b1726-114">No hay ningún valor "sin definir".</span><span class="sxs-lookup"><span data-stu-id="b1726-114">There is no "undefined" value.</span></span> <span data-ttu-id="b1726-115">En muchas aplicaciones de programación, especialmente en las interacciones de las bases de datos, el valor de una variable puede ser indefinido o incluso no estar disponible.</span><span class="sxs-lookup"><span data-stu-id="b1726-115">In many programming applications, most notably database interactions, a variable value can be undefined or missing.</span></span> <span data-ttu-id="b1726-116">Por ejemplo, un campo de una base de datos puede contener los valores true o false, pero también puede no contener ningún valor.</span><span class="sxs-lookup"><span data-stu-id="b1726-116">For example, a field in a database may contain the values true or false, or it may contain no value at all.</span></span> <span data-ttu-id="b1726-117">En tal caso, se usa un tipo `Nullable<bool>`.</span><span class="sxs-lookup"><span data-stu-id="b1726-117">You use a `Nullable<bool>` type in that case.</span></span>

<span data-ttu-id="b1726-118">Los tipos de valor que admiten valores NULL tienen las características siguientes:</span><span class="sxs-lookup"><span data-stu-id="b1726-118">Nullable value types have the following characteristics:</span></span>

- <span data-ttu-id="b1726-119">Los tipos de valor que admiten valores NULL representan variables de tipo de valor a las que se puede asignar el valor `null`.</span><span class="sxs-lookup"><span data-stu-id="b1726-119">Nullable value types represent value-type variables that can be assigned the `null` value.</span></span>

- <span data-ttu-id="b1726-120">La sintaxis `T?` es una abreviatura de `Nullable<T>`.</span><span class="sxs-lookup"><span data-stu-id="b1726-120">The syntax `T?` is shorthand for `Nullable<T>`.</span></span> <span data-ttu-id="b1726-121">Las dos formas son intercambiables.</span><span class="sxs-lookup"><span data-stu-id="b1726-121">The two forms are interchangeable.</span></span>

- <span data-ttu-id="b1726-122">Asigne un valor a un tipo de valor que admite valores NULL tal como lo haría para un tipo de valor subyacente: `int? x = 10;` o `double? d = 4.108;`.</span><span class="sxs-lookup"><span data-stu-id="b1726-122">Assign a value to a nullable value type just as you would for an underlying value type: `int? x = 10;` or `double? d = 4.108;`.</span></span> <span data-ttu-id="b1726-123">También puede asignar el valor `null`: `int? x = null;`.</span><span class="sxs-lookup"><span data-stu-id="b1726-123">You also can assign the `null` value: `int? x = null;`.</span></span>

- <span data-ttu-id="b1726-124">Use las propiedades de solo lectura <xref:System.Nullable%601.HasValue%2A?displayProperty=nameWithType> y <xref:System.Nullable%601.Value%2A?displayProperty=nameWithType> para probar si hay valores NULL y recuperar el valor, como se muestra en el ejemplo siguiente: `if (x.HasValue) y = x.Value;`</span><span class="sxs-lookup"><span data-stu-id="b1726-124">Use the <xref:System.Nullable%601.HasValue%2A?displayProperty=nameWithType> and <xref:System.Nullable%601.Value%2A?displayProperty=nameWithType> readonly properties to test for null and retrieve the value, as shown in the following example: `if (x.HasValue) y = x.Value;`</span></span>

  - <span data-ttu-id="b1726-125">La propiedad <xref:System.Nullable%601.HasValue%2A> devuelve `true` si la variable contiene un valor, o `false` si es `null`.</span><span class="sxs-lookup"><span data-stu-id="b1726-125">The <xref:System.Nullable%601.HasValue%2A> property returns `true` if the variable contains a value, or `false` if it's `null`.</span></span>

  - <span data-ttu-id="b1726-126">La propiedad <xref:System.Nullable%601.Value%2A> devuelve un valor si <xref:System.Nullable%601.HasValue%2A> devuelve `true`.</span><span class="sxs-lookup"><span data-stu-id="b1726-126">The <xref:System.Nullable%601.Value%2A> property returns a value if <xref:System.Nullable%601.HasValue%2A> returns `true`.</span></span> <span data-ttu-id="b1726-127">De lo contrario, se produce una excepción <xref:System.InvalidOperationException>.</span><span class="sxs-lookup"><span data-stu-id="b1726-127">Otherwise, an <xref:System.InvalidOperationException> is thrown.</span></span>

- <span data-ttu-id="b1726-128">También puede usar los operadores `==` y `!=` con un tipo de valor que admite valores NULL, como se muestra en el ejemplo siguiente: `if (x != null) y = x.Value;`.</span><span class="sxs-lookup"><span data-stu-id="b1726-128">You can also use the `==` and `!=` operators with a nullable value type, as shown in the following example: `if (x != null) y = x.Value;`.</span></span> <span data-ttu-id="b1726-129">Si `a` y `b` son ambos NULL, `a == b` se evalúa como `true`.</span><span class="sxs-lookup"><span data-stu-id="b1726-129">If `a` and `b` are both null, `a == b` evaluates to `true`.</span></span>

- <span data-ttu-id="b1726-130">A partir de C# 7.0, se puede usar la [coincidencia de patrones](../../pattern-matching.md#the-is-type-pattern-expression) tanto para examinar como para obtener un valor de un tipo que acepta valores NULL: `if (x is int valueOfX) y = valueOfX;`.</span><span class="sxs-lookup"><span data-stu-id="b1726-130">Beginning with C# 7.0, you can use [pattern matching](../../pattern-matching.md#the-is-type-pattern-expression) to both examine and get a value of a nullable type: `if (x is int valueOfX) y = valueOfX;`.</span></span>

- <span data-ttu-id="b1726-131">El valor predeterminado de `T?` es una instancia cuya propiedad <xref:System.Nullable%601.HasValue%2A> devuelve `false`.</span><span class="sxs-lookup"><span data-stu-id="b1726-131">The default value of `T?` is an instance whose <xref:System.Nullable%601.HasValue%2A> property returns `false`.</span></span>

- <span data-ttu-id="b1726-132">Use el método <xref:System.Nullable%601.GetValueOrDefault> para devolver el valor asignado, o bien el [valor predeterminado](../../language-reference/keywords/default-values-table.md) del tipo subyacente si el valor es `null`.</span><span class="sxs-lookup"><span data-stu-id="b1726-132">Use the <xref:System.Nullable%601.GetValueOrDefault> method to return either the assigned value, or the [default](../../language-reference/keywords/default-values-table.md) value of the underlying value type if the value of the nullable type is `null`.</span></span>

- <span data-ttu-id="b1726-133">Use el método <xref:System.Nullable%601.GetValueOrDefault(%600)> para devolver el valor asignado, o bien el valor predeterminado proporcionado si el valor es `null`.</span><span class="sxs-lookup"><span data-stu-id="b1726-133">Use the <xref:System.Nullable%601.GetValueOrDefault(%600)> method to return either the assigned value, or the provided default value if the value of the nullable type is `null`.</span></span>

- <span data-ttu-id="b1726-134">Use el [operador de fusión NULL](../../language-reference/operators/null-coalescing-operator.md) (`??`) para asignar un valor a una variable de un tipo de valor subyacente basado en un valor que admite NULL: `int? x = null; int y = x ?? -1;`.</span><span class="sxs-lookup"><span data-stu-id="b1726-134">Use the [null-coalescing operator](../../language-reference/operators/null-coalescing-operator.md), `??`, to assign a value to a variable of an underlying value type based on a nullable-type value: `int? x = null; int y = x ?? -1;`.</span></span> <span data-ttu-id="b1726-135">En el ejemplo, dado que `x` es `null`, el valor resultante de `y` es `-1`.</span><span class="sxs-lookup"><span data-stu-id="b1726-135">In the example, since `x` is `null`, the result value of `y` is `-1`.</span></span>

- <span data-ttu-id="b1726-136">Si se establece una conversión definida por el usuario entre dos tipos de valor, esa misma conversión también se puede usar con los correspondientes tipos que admiten un valor NULL.</span><span class="sxs-lookup"><span data-stu-id="b1726-136">If a user-defined conversion is defined between two value types, the same conversion can also be used with the corresponding nullable types.</span></span>

- <span data-ttu-id="b1726-137">No se permite la anidación de tipos de valor que admiten un valor NULL.</span><span class="sxs-lookup"><span data-stu-id="b1726-137">Nested nullable value types are not allowed.</span></span> <span data-ttu-id="b1726-138">La línea siguiente no se compilará: `Nullable<Nullable<int>> n;`</span><span class="sxs-lookup"><span data-stu-id="b1726-138">The following line doesn't compile: `Nullable<Nullable<int>> n;`</span></span>
