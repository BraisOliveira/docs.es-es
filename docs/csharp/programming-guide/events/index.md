## <a name="events-overview"></a><span data-ttu-id="7cf72-109">Información general sobre eventos</span><span class="sxs-lookup"><span data-stu-id="7cf72-109">Events Overview</span></span>  
 <span data-ttu-id="7cf72-110">Los eventos tienen las siguientes propiedades:</span><span class="sxs-lookup"><span data-stu-id="7cf72-110">Events have the following properties:</span></span>  
  
- <span data-ttu-id="7cf72-111">El publicador determina el momento en el que se genera un evento; los suscriptores determinan la acción que se lleva a cabo en respuesta al evento.</span><span class="sxs-lookup"><span data-stu-id="7cf72-111">The publisher determines when an event is raised; the subscribers determine what action is taken in response to the event.</span></span>  
  
- <span data-ttu-id="7cf72-112">Un evento puede tener varios suscriptores.</span><span class="sxs-lookup"><span data-stu-id="7cf72-112">An event can have multiple subscribers.</span></span> <span data-ttu-id="7cf72-113">Un suscriptor puede controlar varios eventos de varios publicadores.</span><span class="sxs-lookup"><span data-stu-id="7cf72-113">A subscriber can handle multiple events from multiple publishers.</span></span>  
  
- <span data-ttu-id="7cf72-114">Nunca se generan eventos que no tienen suscriptores.</span><span class="sxs-lookup"><span data-stu-id="7cf72-114">Events that have no subscribers are never raised.</span></span>  
  
- <span data-ttu-id="7cf72-115">Los eventos se suelen usar para indicar acciones del usuario, como los clics de los botones o las selecciones de menú en las interfaces gráficas de usuario.</span><span class="sxs-lookup"><span data-stu-id="7cf72-115">Events are typically used to signal user actions such as button clicks or menu selections in graphical user interfaces.</span></span>  
  
- <span data-ttu-id="7cf72-116">Cuando un evento tiene varios suscriptores, los controladores de eventos se invocan sincrónicamente cuando se genera un evento.</span><span class="sxs-lookup"><span data-stu-id="7cf72-116">When an event has multiple subscribers, the event handlers are invoked synchronously when an event is raised.</span></span> <span data-ttu-id="7cf72-117">Para invocar eventos de forma asincrónica, consulte [Calling Synchronous Methods Asynchronously](../../../../docs/standard/asynchronous-programming-patterns/calling-synchronous-methods-asynchronously.md).</span><span class="sxs-lookup"><span data-stu-id="7cf72-117">To invoke events asynchronously, see [Calling Synchronous Methods Asynchronously](../../../../docs/standard/asynchronous-programming-patterns/calling-synchronous-methods-asynchronously.md).</span></span>  
  
- <span data-ttu-id="7cf72-118">En la biblioteca de clases de .NET Framework, los eventos se basan en el delegado <xref:System.EventHandler> y en la clase base <xref:System.EventArgs>.</span><span class="sxs-lookup"><span data-stu-id="7cf72-118">In the .NET Framework class library, events are based on the <xref:System.EventHandler> delegate and the <xref:System.EventArgs> base class.</span></span>
