# <a name="classification-of-standard-query-operators-by-manner-of-execution-c"></a><span data-ttu-id="5121b-102">Clasificación de operadores de consulta estándar por modo de ejecución (C#)</span><span class="sxs-lookup"><span data-stu-id="5121b-102">Classification of Standard Query Operators by Manner of Execution (C#)</span></span>
<span data-ttu-id="5121b-103">Las implementaciones de LINQ to Objects de los métodos de operador de consulta estándar se ejecutan de una de dos formas principales: inmediata o aplazada.</span><span class="sxs-lookup"><span data-stu-id="5121b-103">The LINQ to Objects implementations of the standard query operator methods execute in one of two main ways: immediate or deferred.</span></span> <span data-ttu-id="5121b-104">Los operadores de consulta que usan la ejecución aplazada pueden dividirse además en dos categorías: de streaming y de no streaming.</span><span class="sxs-lookup"><span data-stu-id="5121b-104">The query operators that use deferred execution can be additionally divided into two categories: streaming and non-streaming.</span></span> <span data-ttu-id="5121b-105">Si sabe cómo se ejecutan los diferentes operadores de consulta, puede servirle para entender los resultados que se obtienen de una consulta determinada.</span><span class="sxs-lookup"><span data-stu-id="5121b-105">If you know how the different query operators execute, it may help you understand the results that you get from a given query.</span></span> <span data-ttu-id="5121b-106">Esto es especialmente cierto si se está cambiando el origen de datos o si se está creando una consulta sobre otra.</span><span class="sxs-lookup"><span data-stu-id="5121b-106">This is especially true if the data source is changing or if you are building a query on top of another query.</span></span> <span data-ttu-id="5121b-107">En este tema se clasifican los operadores de consulta estándar según su modo de ejecución.</span><span class="sxs-lookup"><span data-stu-id="5121b-107">This topic classifies the standard query operators according to their manner of execution.</span></span>  
  
## <a name="manners-of-execution"></a><span data-ttu-id="5121b-108">Modos de ejecución</span><span class="sxs-lookup"><span data-stu-id="5121b-108">Manners of Execution</span></span>  
  
### <a name="immediate"></a><span data-ttu-id="5121b-109">Inmediato</span><span class="sxs-lookup"><span data-stu-id="5121b-109">Immediate</span></span>  
 <span data-ttu-id="5121b-110">La ejecución inmediata significa que se lee el origen de datos y la operación se realiza en el punto en el código donde se declara la consulta.</span><span class="sxs-lookup"><span data-stu-id="5121b-110">Immediate execution means that the data source is read and the operation is performed at the point in the code where the query is declared.</span></span> <span data-ttu-id="5121b-111">Todos los operadores de consulta estándar que devuelven un resultado único no enumerable se ejecutan de manera inmediata.</span><span class="sxs-lookup"><span data-stu-id="5121b-111">All the standard query operators that return a single, non-enumerable result execute immediately.</span></span>  
  
### <a name="deferred"></a><span data-ttu-id="5121b-112">Aplazada</span><span class="sxs-lookup"><span data-stu-id="5121b-112">Deferred</span></span>  
 <span data-ttu-id="5121b-113">La ejecución aplazada significa que la operación no se realiza en el punto en el código donde se declara la consulta.</span><span class="sxs-lookup"><span data-stu-id="5121b-113">Deferred execution means that the operation is not performed at the point in the code where the query is declared.</span></span> <span data-ttu-id="5121b-114">La operación se realiza solo cuando se enumera la variable de consulta, por ejemplo, mediante una instrucción `foreach`.</span><span class="sxs-lookup"><span data-stu-id="5121b-114">The operation is performed only when the query variable is enumerated, for example by using a `foreach` statement.</span></span> <span data-ttu-id="5121b-115">Esto significa que los resultados de ejecutar la consulta dependen del contenido del origen de datos cuando se ejecuta la consulta en lugar de cuando se define la consulta.</span><span class="sxs-lookup"><span data-stu-id="5121b-115">This means that the results of executing the query depend on the contents of the data source when the query is executed rather than when the query is defined.</span></span> <span data-ttu-id="5121b-116">Si la variable de consulta se enumera varias veces, es posible que los resultados difieran cada vez.</span><span class="sxs-lookup"><span data-stu-id="5121b-116">If the query variable is enumerated multiple times, the results might differ every time.</span></span> <span data-ttu-id="5121b-117">Casi todos los operadores de consulta estándar cuyo tipo de valor devuelto es <xref:System.Collections.Generic.IEnumerable%601> o <xref:System.Linq.IOrderedEnumerable%601> se ejecutan de una manera diferida.</span><span class="sxs-lookup"><span data-stu-id="5121b-117">Almost all the standard query operators whose return type is <xref:System.Collections.Generic.IEnumerable%601> or <xref:System.Linq.IOrderedEnumerable%601> execute in a deferred manner.</span></span>  
  
 <span data-ttu-id="5121b-118">Los operadores de consulta que usan la ejecución aplazada pueden clasificarse además como de streaming o de no streaming.</span><span class="sxs-lookup"><span data-stu-id="5121b-118">Query operators that use deferred execution can be additionally classified as streaming or non-streaming.</span></span>  
  
#### <a name="streaming"></a><span data-ttu-id="5121b-119">Streaming</span><span class="sxs-lookup"><span data-stu-id="5121b-119">Streaming</span></span>  
 <span data-ttu-id="5121b-120">Los operadores de streaming no deben leer todos los datos de origen antes de que produzcan elementos.</span><span class="sxs-lookup"><span data-stu-id="5121b-120">Streaming operators do not have to read all the source data before they yield elements.</span></span> <span data-ttu-id="5121b-121">En el momento de la ejecución, un operador de streaming realiza su operación en cada elemento de origen mientras se lee y proporciona el elemento si es necesario.</span><span class="sxs-lookup"><span data-stu-id="5121b-121">At the time of execution, a streaming operator performs its operation on each source element as it is read and yields the element if appropriate.</span></span> <span data-ttu-id="5121b-122">Un operador de streaming continúa leyendo los elementos de origen hasta que se puede generar un elemento de resultado.</span><span class="sxs-lookup"><span data-stu-id="5121b-122">A streaming operator continues to read source elements until a result element can be produced.</span></span> <span data-ttu-id="5121b-123">Esto significa que es posible leer más de un elemento de origen para generar un elemento de resultado.</span><span class="sxs-lookup"><span data-stu-id="5121b-123">This means that more than one source element might be read to produce one result element.</span></span>  
  
#### <a name="non-streaming"></a><span data-ttu-id="5121b-124">No streaming</span><span class="sxs-lookup"><span data-stu-id="5121b-124">Non-Streaming</span></span>  
 <span data-ttu-id="5121b-125">Los operadores de no streaming deben leer todos los datos de origen antes de poder proporcionar un elemento de resultado.</span><span class="sxs-lookup"><span data-stu-id="5121b-125">Non-streaming operators must read all the source data before they can yield a result element.</span></span> <span data-ttu-id="5121b-126">Las operaciones como la ordenación o la agrupación pertenecen a esta categoría.</span><span class="sxs-lookup"><span data-stu-id="5121b-126">Operations such as sorting or grouping fall into this category.</span></span> <span data-ttu-id="5121b-127">En tiempo de ejecución, los operadores de consulta de no streaming leen todos los datos de origen, los colocan en una estructura de datos, realizan la operación y proporcionan los elementos resultantes.</span><span class="sxs-lookup"><span data-stu-id="5121b-127">At the time of execution, non-streaming query operators read all the source data, put it into a data structure, perform the operation, and yield the resulting elements.</span></span>
