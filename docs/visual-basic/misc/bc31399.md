# <a name="class-classname-has-no-accessible-sub-new-and-cannot-be-inherited"></a><span data-ttu-id="09559-102">La clase\<classname >' no tiene 'Sub New' accesible y no se puede heredar</span><span class="sxs-lookup"><span data-stu-id="09559-102">Class '\<classname>' has no accessible 'Sub New' and cannot be inherited</span></span>
<span data-ttu-id="09559-103">Una clase utiliza un elemento [Inherits Statement](../../visual-basic/language-reference/statements/inherits-statement.md) para especificar una clase base, pero no puede acceder a ningún constructor de la clase base deseada.</span><span class="sxs-lookup"><span data-stu-id="09559-103">A class uses an [Inherits Statement](../../visual-basic/language-reference/statements/inherits-statement.md) to specify a base class, but it cannot access any constructor on the intended base class.</span></span>  
  
 <span data-ttu-id="09559-104">Esto puede ocurrir si la clase base deseada no tiene constructores o si tiene constructores con niveles de acceso que impiden el acceso desde otra clase.</span><span class="sxs-lookup"><span data-stu-id="09559-104">This can happen if the intended base class has no constructors or if it has constructors with access levels that prevent access from another class.</span></span>  
  
 <span data-ttu-id="09559-105">Cuando se hereda de una clase, su constructor debería llamar el constructor de clase base mediante [MyBase](~/docs/visual-basic/programming-guide/program-structure/me-my-mybase-and-myclass.md#mybase).</span><span class="sxs-lookup"><span data-stu-id="09559-105">When you inherit a class, your constructor should call the base class constructor using [MyBase](~/docs/visual-basic/programming-guide/program-structure/me-my-mybase-and-myclass.md#mybase).</span></span> <span data-ttu-id="09559-106">Si no realiza esta llamada o ni siquiera escribe un constructor explícito, Visual Basic genera un constructor implícito que llama a `MyBase.New()`.</span><span class="sxs-lookup"><span data-stu-id="09559-106">If you do not make this call, or if you do not even write an explicit constructor, Visual Basic generates an implicit constructor that calls `MyBase.New()`.</span></span>  
  
 <span data-ttu-id="09559-107">**Identificador de error:** BC31399</span><span class="sxs-lookup"><span data-stu-id="09559-107">**Error ID:** BC31399</span></span>  
  
## <a name="to-correct-this-error"></a><span data-ttu-id="09559-108">Para corregir este error</span><span class="sxs-lookup"><span data-stu-id="09559-108">To correct this error</span></span>  
  
1. <span data-ttu-id="09559-109">Si tiene control de código fuente sobre la clase base deseada, cambie el nivel de acceso de al menos uno de sus constructores para que otra clase pueda acceder a ellos.</span><span class="sxs-lookup"><span data-stu-id="09559-109">If you have source control over the intended base class, then change the access level of at least one of its constructors so that another class can access them.</span></span>  
  
2. <span data-ttu-id="09559-110">Si no puede cambiar los niveles de acceso de los constructores de la clase base deseada, herede de una clase diferente o no herede absolutamente nada.</span><span class="sxs-lookup"><span data-stu-id="09559-110">If you cannot change the access levels of the intended base class constructors, then inherit from a different class or not at all.</span></span>
