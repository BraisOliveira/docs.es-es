# <a name="cannot-implement-interfacename1membername-because-its-implementation-could-conflict-with-the-implementation-for-interfacename2membername-for-some-type-arguments"></a><span data-ttu-id="c1804-102">No se puede implementar '\<interfacename1 >.\< MemberName >' porque su implementación podría entrar en conflicto con la implementación de '\<interfacename2 >.\< MemberName >' para algunos argumentos de tipo</span><span class="sxs-lookup"><span data-stu-id="c1804-102">Cannot implement '\<interfacename1>.\<membername>' because its implementation could conflict with the implementation for '\<interfacename2>.\<membername>' for some type arguments</span></span>
<span data-ttu-id="c1804-103">Una clase implementa más de una interfaz genérica, una de las cuales se hereda de la otra, y dos implementaciones de un miembro de interfaz podría entrar en conflicto para ciertos valores de argumentos de tipo.</span><span class="sxs-lookup"><span data-stu-id="c1804-103">A class implements more than one generic interface, one of which inherits from another, and two implementations of an interface member could conflict for certain values of type arguments.</span></span>  
  
 <span data-ttu-id="c1804-104">Las instrucciones siguientes pueden generar este error.</span><span class="sxs-lookup"><span data-stu-id="c1804-104">The following statements can generate this error.</span></span>  
  
```  
Public Interface iFace1(Of t)  
    Sub testSub()  
End Interface  
Public Interface iFace2(Of u)  
    Inherits iFace1(Of u)  
End Interface  
Public Class testClass(Of y, z)  
    Implements iFace1(Of y), iFace2(Of z)  
    Public Sub testSuby() Implements iFace1(Of y).testSub  
    End Sub  
    Public Sub testSubz() Implements iFace1(Of z).testSub  
    End Sub  
End Class  
```  
  
 <span data-ttu-id="c1804-105">Puesto que `iFace2` hereda de `iFace1` usando su propio parámetro de tipo (`u`), `testClass` implementaría dos versiones de `iFace1.testSub` con firmas idénticas si se pasa el mismo argumento de tipo a `y` y `z`.</span><span class="sxs-lookup"><span data-stu-id="c1804-105">Because `iFace2` inherits from `iFace1` using its own type parameter (`u`), `testClass` would implement two versions of `iFace1.testSub` with identical signatures if the same type argument were passed to `y` and `z`.</span></span> <span data-ttu-id="c1804-106">Esto produciría ambigüedad sobre a qué versión se debe acceder.</span><span class="sxs-lookup"><span data-stu-id="c1804-106">This would produce an ambiguity about which version to access.</span></span>  
  
 <span data-ttu-id="c1804-107">**Identificador de error:** BC32125</span><span class="sxs-lookup"><span data-stu-id="c1804-107">**Error ID:** BC32125</span></span>  
  
## <a name="to-correct-this-error"></a><span data-ttu-id="c1804-108">Para corregir este error</span><span class="sxs-lookup"><span data-stu-id="c1804-108">To correct this error</span></span>  
  
- <span data-ttu-id="c1804-109">Cambie la estructura de herencia de las interfaces para que `iFace1` no se pueda implementar con dos argumentos de tipo diferente.</span><span class="sxs-lookup"><span data-stu-id="c1804-109">Change the inheritance structure of the interfaces so that `iFace1` could not be implemented with two different type arguments.</span></span>  
  
     <span data-ttu-id="c1804-110">-o bien-</span><span class="sxs-lookup"><span data-stu-id="c1804-110">-or-</span></span>  
  
- <span data-ttu-id="c1804-111">Quite de la instrucción `Implements` las interfaces que produzcan el conflicto de implementación.</span><span class="sxs-lookup"><span data-stu-id="c1804-111">Remove from the `Implements` statement one of the interfaces resulting in the implementation conflict.</span></span>
