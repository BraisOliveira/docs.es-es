# <a name="cannot-implement-interface-interfacename1-because-its-implementation-could-conflict-with-the-implementation-of-another-implemented-interface-interfacename2-for-some-type-arguments"></a><span data-ttu-id="11956-102">No se puede implementar la interfaz '\<interfacename1 >' porque su implementación podría entrar en conflicto con la implementación de otra interfaz implementada '\<interfacename2 >' para algunos argumentos de tipo</span><span class="sxs-lookup"><span data-stu-id="11956-102">Cannot implement interface '\<interfacename1>' because its implementation could conflict with the implementation of another implemented interface '\<interfacename2>' for some type arguments</span></span>
<span data-ttu-id="11956-103">Una declaración de clase incluye una instrucción `Implements` que especifica dos o más interfaces, pero al menos una de las interfaces es genérica y dos de las implementaciones podrían entrar en conflicto para determinados valores de argumentos de tipo.</span><span class="sxs-lookup"><span data-stu-id="11956-103">A class declaration includes an `Implements` statement that specifies two or more interfaces, but at least one of the interfaces is generic and two of the implementations could conflict for certain values of type arguments.</span></span>  
  
 <span data-ttu-id="11956-104">Las instrucciones siguientes pueden generar este error.</span><span class="sxs-lookup"><span data-stu-id="11956-104">The following statements can generate this error.</span></span>  
  
```  
Public Interface iFace1  
    Sub testSub(ByVal arg As String)  
End Interface  
Public Interface iFace2(Of t)  
    Sub testSub(ByVal arg As t)  
End Interface  
Public Class testClass  
    Implements iFace1, iFace2(Of String)  
End Class  
```  
  
 <span data-ttu-id="11956-105">Dado que `iFace2` se construye usando `String`, `testClass` debe implementar dos versiones de `testSub` con firmas idénticas.</span><span class="sxs-lookup"><span data-stu-id="11956-105">Because `iFace2` is constructed using `String`, `testClass` must implement two versions of `testSub` with identical signatures.</span></span> <span data-ttu-id="11956-106">Esto produciría ambigüedad sobre a qué versión se debe acceder.</span><span class="sxs-lookup"><span data-stu-id="11956-106">Doing so would produce an ambiguity about which version to access.</span></span>  
  
 <span data-ttu-id="11956-107">**Identificador de error:** BC32072</span><span class="sxs-lookup"><span data-stu-id="11956-107">**Error ID:** BC32072</span></span>  
  
## <a name="to-correct-this-error"></a><span data-ttu-id="11956-108">Para corregir este error</span><span class="sxs-lookup"><span data-stu-id="11956-108">To correct this error</span></span>  
  
- <span data-ttu-id="11956-109">Cambie el argumento de tipo proporcionado a la interfaz genérica para que no haya ningún conflicto.</span><span class="sxs-lookup"><span data-stu-id="11956-109">Change the type argument supplied to the generic interface so that there is no conflict.</span></span>  
  
     <span data-ttu-id="11956-110">-o bien-</span><span class="sxs-lookup"><span data-stu-id="11956-110">-or-</span></span>  
  
- <span data-ttu-id="11956-111">Quite de la instrucción `Implements` las interfaces que produzcan el conflicto de implementación.</span><span class="sxs-lookup"><span data-stu-id="11956-111">Remove from the `Implements` statement one of the interfaces resulting in the implementation conflict.</span></span>
