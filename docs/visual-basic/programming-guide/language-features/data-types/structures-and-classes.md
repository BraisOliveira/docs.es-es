# <a name="structures-and-classes-visual-basic"></a><span data-ttu-id="ac50b-102">Estructuras y clases (Visual Basic)</span><span class="sxs-lookup"><span data-stu-id="ac50b-102">Structures and Classes (Visual Basic)</span></span>
<span data-ttu-id="ac50b-103">Visual Basic unifica la sintaxis para las estructuras y clases, con lo que ambas entidades admiten la mayoría de las mismas características.</span><span class="sxs-lookup"><span data-stu-id="ac50b-103">Visual Basic unifies the syntax for structures and classes, with the result that both entities support most of the same features.</span></span> <span data-ttu-id="ac50b-104">Sin embargo, también hay diferencias importantes entre las clases y estructuras.</span><span class="sxs-lookup"><span data-stu-id="ac50b-104">However, there are also important differences between structures and classes.</span></span>  
  
 <span data-ttu-id="ac50b-105">Las clases tienen la ventaja de ser tipos de referencia, pasando una referencia es más eficaz que se pasa una variable de estructura con todos sus datos.</span><span class="sxs-lookup"><span data-stu-id="ac50b-105">Classes have the advantage of being reference types — passing a reference is more efficient than passing a structure variable with all its data.</span></span> <span data-ttu-id="ac50b-106">Por otro lado, las estructuras no requieren la asignación de memoria del montón global.</span><span class="sxs-lookup"><span data-stu-id="ac50b-106">On the other hand, structures do not require allocation of memory on the global heap.</span></span>

## <a name="similarities"></a><span data-ttu-id="ac50b-109">Similitudes</span><span class="sxs-lookup"><span data-stu-id="ac50b-109">Similarities</span></span>  
 <span data-ttu-id="ac50b-110">Estructuras y clases son similares en los siguientes aspectos:</span><span class="sxs-lookup"><span data-stu-id="ac50b-110">Structures and classes are similar in the following respects:</span></span>  
  
- <span data-ttu-id="ac50b-111">Ambos son *contenedor* tipos, lo que significa que contienen otros tipos como miembros.</span><span class="sxs-lookup"><span data-stu-id="ac50b-111">Both are *container* types, meaning that they contain other types as members.</span></span>  
  
- <span data-ttu-id="ac50b-112">Ambas tienen miembros, que pueden incluir constructores, métodos, propiedades, campos, constantes, enumeraciones, eventos y controladores de eventos.</span><span class="sxs-lookup"><span data-stu-id="ac50b-112">Both have members, which can include constructors, methods, properties, fields, constants, enumerations, events, and event handlers.</span></span> <span data-ttu-id="ac50b-113">Sin embargo, no confunda estos miembros con el declarado *elementos* de una estructura.</span><span class="sxs-lookup"><span data-stu-id="ac50b-113">However, do not confuse these members with the declared *elements* of a structure.</span></span>  
  
- <span data-ttu-id="ac50b-114">Niveles de acceso pueden haber individualizados los miembros de ambos.</span><span class="sxs-lookup"><span data-stu-id="ac50b-114">Members of both can have individualized access levels.</span></span> <span data-ttu-id="ac50b-115">Por ejemplo, se puede declarar un miembro `Public` y otra `Private`.</span><span class="sxs-lookup"><span data-stu-id="ac50b-115">For example, one member can be declared `Public` and another `Private`.</span></span>  
  
- <span data-ttu-id="ac50b-116">Ambos pueden implementar interfaces.</span><span class="sxs-lookup"><span data-stu-id="ac50b-116">Both can implement interfaces.</span></span>  
  
- <span data-ttu-id="ac50b-117">Ambos pueden tener constructores compartidos, con o sin parámetros.</span><span class="sxs-lookup"><span data-stu-id="ac50b-117">Both can have shared constructors, with or without parameters.</span></span>  
  
- <span data-ttu-id="ac50b-118">Ambas pueden exponer una *propiedad predeterminada*, siempre que la propiedad tiene al menos un parámetro.</span><span class="sxs-lookup"><span data-stu-id="ac50b-118">Both can expose a *default property*, provided that property takes at least one parameter.</span></span>  
  
- <span data-ttu-id="ac50b-119">Ambos pueden declarar y provocar eventos, y ambas pueden declarar a los delegados.</span><span class="sxs-lookup"><span data-stu-id="ac50b-119">Both can declare and raise events, and both can declare delegates.</span></span>  
  
## <a name="differences"></a><span data-ttu-id="ac50b-120">Diferencias</span><span class="sxs-lookup"><span data-stu-id="ac50b-120">Differences</span></span>  
 <span data-ttu-id="ac50b-121">Estructuras y clases difieren en las siguientes indicaciones:</span><span class="sxs-lookup"><span data-stu-id="ac50b-121">Structures and classes differ in the following particulars:</span></span>  
  
- <span data-ttu-id="ac50b-122">Las estructuras son *los tipos de valor*; las clases son *hacen referencia a tipos*.</span><span class="sxs-lookup"><span data-stu-id="ac50b-122">Structures are *value types*; classes are *reference types*.</span></span> <span data-ttu-id="ac50b-123">Una variable de un tipo de estructura contiene datos de la estructura, en lugar que contiene una referencia a los datos como un tipo de clase.</span><span class="sxs-lookup"><span data-stu-id="ac50b-123">A variable of a structure type contains the structure's data, rather than containing a reference to the data as a class type does.</span></span>  
  
- <span data-ttu-id="ac50b-124">Estructuras de utilizar asignación de la pila; las clases utilizan la asignación del montón.</span><span class="sxs-lookup"><span data-stu-id="ac50b-124">Structures use stack allocation; classes use heap allocation.</span></span>  
  
- <span data-ttu-id="ac50b-125">Todos los elementos de estructura son `Public` de forma predeterminada; la clase variables y constantes son `Private` de forma predeterminada, mientras que otros miembros de clase son `Public` de forma predeterminada.</span><span class="sxs-lookup"><span data-stu-id="ac50b-125">All structure elements are `Public` by default; class variables and constants are `Private` by default, while other class members are `Public` by default.</span></span> <span data-ttu-id="ac50b-126">Este comportamiento para los miembros de clase proporciona compatibilidad con el sistema de Visual Basic 6.0 de los valores predeterminados.</span><span class="sxs-lookup"><span data-stu-id="ac50b-126">This behavior for class members provides compatibility with the Visual Basic 6.0 system of defaults.</span></span>  
  
- <span data-ttu-id="ac50b-127">Una estructura debe tener al menos no compartido no compartido, o variable no personalizado elemento eventos; una clase puede estar completamente vacía.</span><span class="sxs-lookup"><span data-stu-id="ac50b-127">A structure must have at least one nonshared variable or nonshared, noncustom event element; a class can be completely empty.</span></span>  
  
- <span data-ttu-id="ac50b-128">Elementos de la estructura no se pueden declarar como `Protected`; los miembros de clase pueden.</span><span class="sxs-lookup"><span data-stu-id="ac50b-128">Structure elements cannot be declared as `Protected`; class members can.</span></span>  
  
- <span data-ttu-id="ac50b-129">Un procedimiento de estructura puede controlar eventos solo si es un [Shared](../../../../visual-basic/language-reference/modifiers/shared.md) `Sub` procedimiento y sólo por medio de la [AddHandler (instrucción)](../../../../visual-basic/language-reference/statements/addhandler-statement.md); cualquier procedimiento de clase puede controlar eventos, mediante cualquiera de los [ Controla](../../../../visual-basic/language-reference/statements/handles-clause.md) palabra clave o el `AddHandler` instrucción.</span><span class="sxs-lookup"><span data-stu-id="ac50b-129">A structure procedure can handle events only if it is a [Shared](../../../../visual-basic/language-reference/modifiers/shared.md)`Sub` procedure, and only by means of the [AddHandler Statement](../../../../visual-basic/language-reference/statements/addhandler-statement.md); any class procedure can handle events, using either the [Handles](../../../../visual-basic/language-reference/statements/handles-clause.md) keyword or the `AddHandler` statement.</span></span> <span data-ttu-id="ac50b-130">Para más información, vea [Eventos](../../../../visual-basic/programming-guide/language-features/events/index.md).</span><span class="sxs-lookup"><span data-stu-id="ac50b-130">For more information, see [Events](../../../../visual-basic/programming-guide/language-features/events/index.md).</span></span>  
  
- <span data-ttu-id="ac50b-131">Las declaraciones de variables de estructura no pueden especificar inicializadores o tamaños iniciales para matrices; las declaraciones de variable de clase pueden.</span><span class="sxs-lookup"><span data-stu-id="ac50b-131">Structure variable declarations cannot specify initializers or initial sizes for arrays; class variable declarations can.</span></span>  
  
- <span data-ttu-id="ac50b-132">Las estructuras heredan implícitamente de la <xref:System.ValueType?displayProperty=nameWithType> clase y no puede heredar de cualquier otro tipo; las clases pueden heredar de cualquier clase o clases distinto <xref:System.ValueType?displayProperty=nameWithType>.</span><span class="sxs-lookup"><span data-stu-id="ac50b-132">Structures implicitly inherit from the <xref:System.ValueType?displayProperty=nameWithType> class and cannot inherit from any other type; classes can inherit from any class or classes other than <xref:System.ValueType?displayProperty=nameWithType>.</span></span>  
  
- <span data-ttu-id="ac50b-133">Las estructuras no son heredables; las clases son.</span><span class="sxs-lookup"><span data-stu-id="ac50b-133">Structures are not inheritable; classes are.</span></span>  
  
- <span data-ttu-id="ac50b-134">Nunca se terminan las estructuras de modo que nunca llame common language runtime (CLR) los <xref:System.Object.Finalize%2A> método en una estructura; las clases se terminan por el recolector de elementos no utilizados (GC), que llama a <xref:System.Object.Finalize%2A> en una clase cuando detecta que no hay referencias activas restantes.</span><span class="sxs-lookup"><span data-stu-id="ac50b-134">Structures are never terminated, so the common language runtime (CLR) never calls the <xref:System.Object.Finalize%2A> method on any structure; classes are terminated by the garbage collector (GC), which calls <xref:System.Object.Finalize%2A> on a class when it detects there are no active references remaining.</span></span>  
  
- <span data-ttu-id="ac50b-135">Una estructura no requiere un constructor; realiza una clase.</span><span class="sxs-lookup"><span data-stu-id="ac50b-135">A structure does not require a constructor; a class does.</span></span>  
  
- <span data-ttu-id="ac50b-136">Las estructuras pueden tener constructores no compartidos únicamente si toman parámetros; las clases pueden tenerlos con o sin parámetros.</span><span class="sxs-lookup"><span data-stu-id="ac50b-136">Structures can have nonshared constructors only if they take parameters; classes can have them with or without parameters.</span></span>  
  
 <span data-ttu-id="ac50b-137">Cada estructura tiene un constructor implícito público sin parámetros.</span><span class="sxs-lookup"><span data-stu-id="ac50b-137">Every structure has an implicit public constructor without parameters.</span></span> <span data-ttu-id="ac50b-138">Este constructor inicializa de elementos de datos el de la estructura en sus valores predeterminados.</span><span class="sxs-lookup"><span data-stu-id="ac50b-138">This constructor initializes all the structure's data elements to their default values.</span></span> <span data-ttu-id="ac50b-139">No se puede volver a definir este comportamiento.</span><span class="sxs-lookup"><span data-stu-id="ac50b-139">You cannot redefine this behavior.</span></span>  
  
## <a name="instances-and-variables"></a><span data-ttu-id="ac50b-140">Instancias y Variables</span><span class="sxs-lookup"><span data-stu-id="ac50b-140">Instances and Variables</span></span>  
 <span data-ttu-id="ac50b-141">Dado que las estructuras son tipos de valor, cada variable de estructura permanentemente está enlazado a una instancia de estructura individual.</span><span class="sxs-lookup"><span data-stu-id="ac50b-141">Because structures are value types, each structure variable is permanently bound to an individual structure instance.</span></span> <span data-ttu-id="ac50b-142">Pero las clases son tipos de referencia y una variable de objeto puede hacer referencia a varias instancias de clase en momentos diferentes.</span><span class="sxs-lookup"><span data-stu-id="ac50b-142">But classes are reference types, and an object variable can refer to various class instances at different times.</span></span> <span data-ttu-id="ac50b-143">Esta distinción afecta al uso de estructuras y clases de las maneras siguientes:</span><span class="sxs-lookup"><span data-stu-id="ac50b-143">This distinction affects your usage of structures and classes in the following ways:</span></span>  
  
- <span data-ttu-id="ac50b-144">**Inicialización.**</span><span class="sxs-lookup"><span data-stu-id="ac50b-144">**Initialization.**</span></span> <span data-ttu-id="ac50b-145">Una variable de estructura incluye implícitamente una inicialización de los elementos mediante el constructor sin parámetros de la estructura.</span><span class="sxs-lookup"><span data-stu-id="ac50b-145">A structure variable implicitly includes an initialization of the elements using the structure's parameterless constructor.</span></span> <span data-ttu-id="ac50b-146">Por lo tanto, `Dim s As struct1` es equivalente a `Dim s As struct1 = New struct1()`.</span><span class="sxs-lookup"><span data-stu-id="ac50b-146">Therefore, `Dim s As struct1` is equivalent to `Dim s As struct1 = New struct1()`.</span></span>  
  
- <span data-ttu-id="ac50b-147">**Asignación de Variables.**</span><span class="sxs-lookup"><span data-stu-id="ac50b-147">**Assigning Variables.**</span></span> <span data-ttu-id="ac50b-148">Al asignar una variable de estructura a otro, o pasar una instancia de la estructura a un argumento de procedimiento, se copian los valores actuales de todos los elementos de la variable a la nueva estructura.</span><span class="sxs-lookup"><span data-stu-id="ac50b-148">When you assign one structure variable to another, or pass a structure instance to a procedure argument, the current values of all the variable elements are copied to the new structure.</span></span> <span data-ttu-id="ac50b-149">Al asignar una variable de objeto a otro, o pasar una variable de objeto a un procedimiento, se copia solo el puntero de referencia.</span><span class="sxs-lookup"><span data-stu-id="ac50b-149">When you assign one object variable to another, or pass an object variable to a procedure, only the reference pointer is copied.</span></span>  
  
- <span data-ttu-id="ac50b-150">**Asignar nada.**</span><span class="sxs-lookup"><span data-stu-id="ac50b-150">**Assigning Nothing.**</span></span> <span data-ttu-id="ac50b-151">Puede asignar el valor [nada](../../../../visual-basic/language-reference/nothing.md) a una estructura variable, pero la instancia continúa va a asociar a la variable.</span><span class="sxs-lookup"><span data-stu-id="ac50b-151">You can assign the value [Nothing](../../../../visual-basic/language-reference/nothing.md) to a structure variable, but the instance continues to be associated with the variable.</span></span> <span data-ttu-id="ac50b-152">Todavía puede llamar a sus métodos y tener acceso a sus elementos de datos, aunque se reinicializan elementos variable mediante la asignación.</span><span class="sxs-lookup"><span data-stu-id="ac50b-152">You can still call its methods and access its data elements, although variable elements are reinitialized by the assignment.</span></span>  
  
     <span data-ttu-id="ac50b-153">En cambio, si establece una variable de objeto en `Nothing`, desasocia cualquier instancia de clase y no se puede obtener acceso a todos los miembros a través de la variable hasta que asigne otra instancia.</span><span class="sxs-lookup"><span data-stu-id="ac50b-153">In contrast, if you set an object variable to `Nothing`, you dissociate it from any class instance, and you cannot access any members through the variable until you assign another instance to it.</span></span>  
  
- <span data-ttu-id="ac50b-154">**Varias instancias.**</span><span class="sxs-lookup"><span data-stu-id="ac50b-154">**Multiple Instances.**</span></span> <span data-ttu-id="ac50b-155">Una variable de objeto puede tener distintas instancias de clase asignadas a ella en momentos diferentes, y varias variables de objeto pueden hacer referencia a la misma instancia de clase al mismo tiempo.</span><span class="sxs-lookup"><span data-stu-id="ac50b-155">An object variable can have different class instances assigned to it at different times, and several object variables can refer to the same class instance at the same time.</span></span> <span data-ttu-id="ac50b-156">Cambios en los valores de los miembros de clase afectan a dichos miembros cuando se tiene acceso a través de otra variable que apunta a la misma instancia.</span><span class="sxs-lookup"><span data-stu-id="ac50b-156">Changes you make to the values of class members affect those members when accessed through another variable pointing to the same instance.</span></span>  
  
     <span data-ttu-id="ac50b-157">Elementos de estructura, sin embargo, están aislados dentro de su propia instancia.</span><span class="sxs-lookup"><span data-stu-id="ac50b-157">Structure elements, however, are isolated within their own instance.</span></span> <span data-ttu-id="ac50b-158">No se reflejan los cambios en sus valores en otras variables de estructura, incluso en otras instancias del mismo `Structure` declaración.</span><span class="sxs-lookup"><span data-stu-id="ac50b-158">Changes to their values are not reflected in any other structure variables, even in other instances of the same `Structure` declaration.</span></span>  
  
- <span data-ttu-id="ac50b-159">**Igualdad.**</span><span class="sxs-lookup"><span data-stu-id="ac50b-159">**Equality.**</span></span> <span data-ttu-id="ac50b-160">Comprobar la igualdad de dos estructuras debe realizarse con una elemento por elemento prueba.</span><span class="sxs-lookup"><span data-stu-id="ac50b-160">Equality testing of two structures must be performed with an element-by-element test.</span></span> <span data-ttu-id="ac50b-161">Se pueden comparar dos variables de objeto mediante el <xref:System.Object.Equals%2A> método.</span><span class="sxs-lookup"><span data-stu-id="ac50b-161">Two object variables can be compared using the <xref:System.Object.Equals%2A> method.</span></span> <span data-ttu-id="ac50b-162"><xref:System.Object.Equals%2A> indica si las dos variables apuntan a la misma instancia.</span><span class="sxs-lookup"><span data-stu-id="ac50b-162"><xref:System.Object.Equals%2A> indicates whether the two variables point to the same instance.</span></span>
