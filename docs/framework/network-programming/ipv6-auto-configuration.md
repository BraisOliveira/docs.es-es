# <a name="ipv6-auto-configuration"></a><span data-ttu-id="1d446-102">Configuración automática de IPv6</span><span class="sxs-lookup"><span data-stu-id="1d446-102">IPv6 Auto-Configuration</span></span>
<span data-ttu-id="1d446-103">Un objetivo importante para IPv6 es ser compatible con el nodo Plug and Play.</span><span class="sxs-lookup"><span data-stu-id="1d446-103">One important goal for IPv6 is to support node Plug and Play.</span></span> <span data-ttu-id="1d446-104">Es decir, debería ser posible conectar un nodo en una red IPv6 y hacer que se configurase automáticamente sin intervención humana.</span><span class="sxs-lookup"><span data-stu-id="1d446-104">That is, it should be possible to plug a node into an IPv6 network and have it automatically configured without any human intervention.</span></span>  
  
## <a name="type-of-auto-configuration"></a><span data-ttu-id="1d446-105">Tipo de configuración automática</span><span class="sxs-lookup"><span data-stu-id="1d446-105">Type of Auto-Configuration</span></span>  
 <span data-ttu-id="1d446-106">IPv6 es compatible con los siguientes tipos de configuración automática:</span><span class="sxs-lookup"><span data-stu-id="1d446-106">IPv6 supports the following types of auto-configuration:</span></span>  
  
- <span data-ttu-id="1d446-107">**Configuración automática con estado**.</span><span class="sxs-lookup"><span data-stu-id="1d446-107">**Stateful auto-configuration**.</span></span> <span data-ttu-id="1d446-108">Este tipo de configuración requiere cierto grado de intervención humana, ya que es necesario un servidor de Protocolo de configuración dinámica de host para IPv6 (DHCPv6) para la instalación y la administración de los nodos.</span><span class="sxs-lookup"><span data-stu-id="1d446-108">This type of configuration requires a certain level of human intervention because it needs a Dynamic Host Configuration Protocol for IPv6 (DHCPv6) server for the installation and administration of the nodes.</span></span> <span data-ttu-id="1d446-109">El servidor DHCPv6 mantiene una lista de nodos a los que proporciona información de configuración.</span><span class="sxs-lookup"><span data-stu-id="1d446-109">The DHCPv6 server keeps a list of nodes to which it supplies configuration information.</span></span> <span data-ttu-id="1d446-110">También mantiene información de estado para que el servidor sepa cuánto tiempo se usa cada dirección y cuándo podría estar disponible para su reasignación.</span><span class="sxs-lookup"><span data-stu-id="1d446-110">It also maintains state information so the server knows how long each address is in use, and when it might be available for reassignment.</span></span>  
  
- <span data-ttu-id="1d446-111">**Configuración automática sin estado**.</span><span class="sxs-lookup"><span data-stu-id="1d446-111">**Stateless auto-configuration**.</span></span> <span data-ttu-id="1d446-112">Este tipo de configuración es adecuada para organizaciones pequeñas y usuarios individuales.</span><span class="sxs-lookup"><span data-stu-id="1d446-112">This type of configuration is suitable for small organizations and individuals.</span></span> <span data-ttu-id="1d446-113">En este caso, cada host determina sus direcciones a partir del contenido de los anuncios de enrutador recibidos.</span><span class="sxs-lookup"><span data-stu-id="1d446-113">In this case, each host determines its addresses from the contents of received router advertisements.</span></span> <span data-ttu-id="1d446-114">El uso del estándar IEEE EUI-64 para definir la parte del identificador de red de la dirección permite dar por supuesto que la dirección de host del vínculo es exclusiva.</span><span class="sxs-lookup"><span data-stu-id="1d446-114">Using the IEEE EUI-64 standard to define the network ID portion of the address, it is reasonable to assume the uniqueness of the host address on the link.</span></span>  
  
 <span data-ttu-id="1d446-115">Independientemente de cómo se determine la dirección, el nodo debe comprobar que su dirección potencial es única para el vínculo local.</span><span class="sxs-lookup"><span data-stu-id="1d446-115">Regardless of how the address is determined, the node must verify that its potential address is unique to the local link.</span></span> <span data-ttu-id="1d446-116">Para ello, envía un mensaje de convocatoria de vecino a la dirección potencial.</span><span class="sxs-lookup"><span data-stu-id="1d446-116">This is done by sending a neighbor solicitation message to the potential address.</span></span> <span data-ttu-id="1d446-117">Si el nodo recibe respuesta, sabe que la dirección ya está en uso y debe establecer otra dirección.</span><span class="sxs-lookup"><span data-stu-id="1d446-117">If the node receives any response, it knows that the address is already in use and must determine another address.</span></span>  
  
## <a name="ipv6-mobility"></a><span data-ttu-id="1d446-118">Movilidad de IPv6</span><span class="sxs-lookup"><span data-stu-id="1d446-118">IPv6 Mobility</span></span>  
 <span data-ttu-id="1d446-119">La proliferación de dispositivos móviles ha introducido un requisito nuevo: un dispositivo debe poder cambiar de forma arbitraria las ubicaciones en Internet IPv6 y seguir manteniendo las conexiones existentes.</span><span class="sxs-lookup"><span data-stu-id="1d446-119">The proliferation of mobile devices has introduced a new requirement: A device must be able to arbitrarily change locations on the IPv6 Internet and still maintain existing connections.</span></span> <span data-ttu-id="1d446-120">Para proporcionar esta funcionalidad, se asigna a un nodo móvil una dirección principal en la que siempre se puede establecer contacto con él.</span><span class="sxs-lookup"><span data-stu-id="1d446-120">To provide this functionality, a mobile node is assigned a home address at which it can always be reached.</span></span> <span data-ttu-id="1d446-121">Cuando el nodo móvil se encuentra en su ubicación principal, se conecta con el vínculo principal y usa su dirección principal.</span><span class="sxs-lookup"><span data-stu-id="1d446-121">When the mobile node is at home, it connects to the home link and uses its home address.</span></span> <span data-ttu-id="1d446-122">Cuando el nodo móvil cambia de ubicación, un agente principal, que normalmente es un enrutador, transmite los mensajes entre el nodo móvil y los nodos con los que se comunica.</span><span class="sxs-lookup"><span data-stu-id="1d446-122">When the mobile node is away from home, a home agent, which is usually a router, relays messages between the mobile node and nodes with which it is communicating.</span></span>
