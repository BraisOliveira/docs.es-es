## <a name="port-sharing-architecture"></a><span data-ttu-id="d9fd8-121">Arquitectura de uso compartido de puertos</span><span class="sxs-lookup"><span data-stu-id="d9fd8-121">Port Sharing Architecture</span></span>  
 <span data-ttu-id="d9fd8-122">La arquitectura de uso compartido de puertos en WCF tiene tres componentes principales:</span><span class="sxs-lookup"><span data-stu-id="d9fd8-122">The Port Sharing architecture in WCF has three main components:</span></span>  
  
-   <span data-ttu-id="d9fd8-123">Un proceso de trabajo: Cualquier proceso que se comunica sobre net.tcp:// utilizando los puertos compartidos.</span><span class="sxs-lookup"><span data-stu-id="d9fd8-123">A Worker Process: Any process communicating over net.tcp:// using shared ports.</span></span>  
  
-   <span data-ttu-id="d9fd8-124">El transporte TCP de WCF: Implementa el protocolo net.tcp://.</span><span class="sxs-lookup"><span data-stu-id="d9fd8-124">The WCF TCP transport: Implements the net.tcp:// protocol.</span></span>  
  
-   <span data-ttu-id="d9fd8-125">El servicio de uso compartido de puertos Net.TCP: Permite que varios procesos de trabajo compartir el mismo puerto TCP.</span><span class="sxs-lookup"><span data-stu-id="d9fd8-125">The Net.TCP Port Sharing Service: Allows many worker processes to share the same TCP port.</span></span>  
  
 <span data-ttu-id="d9fd8-126">El servicio de uso compartido de puertos Net.TCP es un servicio de Windows de modo de usuario que acepta conexiones net.tcp:// en nombre de los procesos de trabajo que se conectan a través de él.</span><span class="sxs-lookup"><span data-stu-id="d9fd8-126">The Net.TCP Port Sharing Service is a user-mode Windows service that accepts net.tcp:// connections on behalf of the worker processes that connect through it.</span></span> <span data-ttu-id="d9fd8-127">Cuando una conexión de socket llega, el servicio de uso compartido de puertos inspecciona la secuencia del mensaje entrante para obtener su dirección de destino.</span><span class="sxs-lookup"><span data-stu-id="d9fd8-127">When a socket connection arrives, the port sharing service inspects the incoming message stream to obtain its destination address.</span></span> <span data-ttu-id="d9fd8-128">Basándose en esta dirección, el servicio de uso compartido de puertos puede enrutar el flujo de datos a la aplicación que finalmente lo procesa.</span><span class="sxs-lookup"><span data-stu-id="d9fd8-128">Based on this address, the port sharing service can route the data stream to the application that ultimately processes it.</span></span>  
  
 <span data-ttu-id="d9fd8-129">Cuando un servicio WCF que usa los puertos de net.tcp:// se abre, la infraestructura de transporte TCP de WCF no abre directamente un socket TCP en el proceso de aplicación.</span><span class="sxs-lookup"><span data-stu-id="d9fd8-129">When a WCF service that uses net.tcp:// port sharing opens, the WCF TCP transport infrastructure does not directly open a TCP socket in the application process.</span></span> <span data-ttu-id="d9fd8-130">En su lugar, la infraestructura de transporte registra el Identificador uniforme de recursos (URI) de la dirección base del servicio con el servicio de uso compartido de puertos Net.TCP y espera a que el servicio de uso compartido de puertos escuche mensajes en su nombre.</span><span class="sxs-lookup"><span data-stu-id="d9fd8-130">Instead, the transport infrastructure registers the service’s base address Uniform Resource Identifier (URI) with the Net.TCP Port Sharing Service and waits for the port sharing service to listen for messages on its behalf.</span></span>  <span data-ttu-id="d9fd8-131">El servicio de uso compartido de puertos entrega mensajes direccionados al servicio de la aplicación según van llegando.</span><span class="sxs-lookup"><span data-stu-id="d9fd8-131">The port sharing service dispatches messages addressed to the application service as they arrive.</span></span>
