 <span data-ttu-id="c30cf-108">En un modelo conceptual, se pueden crear jerarquías de herencia en las que un tipo derivado hereda de otro tipo derivado.</span><span class="sxs-lookup"><span data-stu-id="c30cf-108">In a conceptual model you can build inheritance hierarchies in which a derived type inherits from another derived type.</span></span> <span data-ttu-id="c30cf-109">El tipo en la parte superior de la jerarquía (un tipo de la jerarquía que no es un tipo derivado) se denomina la *el tipo de raíz*.</span><span class="sxs-lookup"><span data-stu-id="c30cf-109">The type at the top of the hierarchy (the one type in the hierarchy that is not a derived type) is called the *root type*.</span></span> <span data-ttu-id="c30cf-110">En una jerarquía de herencia, la [clave de entidad](../../../../docs/framework/data/adonet/entity-key.md) debe definirse en el tipo raíz.</span><span class="sxs-lookup"><span data-stu-id="c30cf-110">In an inheritance hierarchy, the [entity key](../../../../docs/framework/data/adonet/entity-key.md) must be defined on the root type.</span></span>  
  
 <span data-ttu-id="c30cf-111">No se pueden crear jerarquías de herencia en las que un tipo derivado hereda de más de un tipo.</span><span class="sxs-lookup"><span data-stu-id="c30cf-111">You cannot build inheritance hierarchies in which a derived type inherits from more than one type.</span></span> <span data-ttu-id="c30cf-112">Por ejemplo, en un modelo conceptual con un tipo de entidad `Book`, se podrían definir tipos derivados `FictionBook` y `NonFictionBook` que hereden de `Book`.</span><span class="sxs-lookup"><span data-stu-id="c30cf-112">For example, in a conceptual model with a `Book` entity type, you could define derived types `FictionBook` and `NonFictionBook` that each inherit from `Book`.</span></span> <span data-ttu-id="c30cf-113">Sin embargo, no se podría definir un tipo que herede de los tipos `FictionBook` y `NonFictionBook`.</span><span class="sxs-lookup"><span data-stu-id="c30cf-113">However, you could not then define a type that inherits from both the `FictionBook` and `NonFictionBook` types.</span></span>
