## <a name="remarks"></a><span data-ttu-id="6eaca-108">Comentarios</span><span class="sxs-lookup"><span data-stu-id="6eaca-108">Remarks</span></span>  
 <span data-ttu-id="6eaca-109">La consulta siguiente genera una lista de productos y una colección de cantidades de la línea de pedidos por cada producto:</span><span class="sxs-lookup"><span data-stu-id="6eaca-109">The following query produces a list of products and a collection of order line quantities per each product:</span></span>

  group by ol.Product as p  
```  
  
 <span data-ttu-id="6eaca-110">Las dos consultas siguientes son semánticamente iguales:</span><span class="sxs-lookup"><span data-stu-id="6eaca-110">The following two queries are semantically equal:</span></span>

  group by ol.Product as p

 <span data-ttu-id="6eaca-111">El operador `GROUPPARTITION` se puede utilizar junto con funciones de agregado definidas por el usuario.</span><span class="sxs-lookup"><span data-stu-id="6eaca-111">The `GROUPPARTITION` operator can be used in conjunction with user-defined aggregate functions.</span></span>

```

```  
  
 <span data-ttu-id="6eaca-115">Con un GROUP BY normal, se ocultan los resultados de la agrupación.</span><span class="sxs-lookup"><span data-stu-id="6eaca-115">With a regular GROUP BY, the results of the grouping are hidden.</span></span> <span data-ttu-id="6eaca-116">Los resultados solo pueden utilizarse en una función de agregado.</span><span class="sxs-lookup"><span data-stu-id="6eaca-116">You can only use the results in an aggregate function.</span></span> <span data-ttu-id="6eaca-117">Para ver los resultados de la agrupación, tiene que poner en correlación los resultados de la agrupación y el conjunto de entrada mediante una subconsulta.</span><span class="sxs-lookup"><span data-stu-id="6eaca-117">In order to see the results of the grouping, you have to correlate the results of the grouping and the input set by using a subquery.</span></span> <span data-ttu-id="6eaca-118">Las dos consultas siguientes son equivalentes:</span><span class="sxs-lookup"><span data-stu-id="6eaca-118">The following two queries are equivalent:</span></span>

```  
  
 <span data-ttu-id="6eaca-119">Tal y como se ve en el ejemplo, el operador agregado GROUPPARTITION facilita la obtención de una referencia al conjunto de entrada después de la agrupación.</span><span class="sxs-lookup"><span data-stu-id="6eaca-119">As seen from the example, the GROUPPARTITION aggregate operator makes it easier to get a reference to the input set after the grouping.</span></span>  
  
 <span data-ttu-id="6eaca-120">El operador GROUPPARTITION puede especificar cualquier expresión [!INCLUDE[esql](../../../../../../includes/esql-md.md)] en la entrada del operador cuando se utiliza el parámetro `expression` .</span><span class="sxs-lookup"><span data-stu-id="6eaca-120">The GROUPPARTITION operator can specify any [!INCLUDE[esql](../../../../../../includes/esql-md.md)] expression in the operator input when you use the `expression` parameter.</span></span>  
  
 <span data-ttu-id="6eaca-121">Por ejemplo todas las expresiones de entrada siguientes a la partición de grupo son válidas:</span><span class="sxs-lookup"><span data-stu-id="6eaca-121">For instance all of the following input expressions to the group partition are valid:</span></span>
