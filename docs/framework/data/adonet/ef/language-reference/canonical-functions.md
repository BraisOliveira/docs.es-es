# <a name="canonical-functions"></a><span data-ttu-id="21489-102">Funciones canónicas</span><span class="sxs-lookup"><span data-stu-id="21489-102">Canonical Functions</span></span>
<span data-ttu-id="21489-103">Esta sección describe las funciones canónicas que son admitidas por todos los proveedores de datos y pueden ser utilizadas por todas las tecnologías de creación de consultas.</span><span class="sxs-lookup"><span data-stu-id="21489-103">This section discusses canonical functions that are supported by all data providers, and can be used by all querying technologies.</span></span> <span data-ttu-id="21489-104">Las funciones canónicas no pueden ser ampliadas por un proveedor.</span><span class="sxs-lookup"><span data-stu-id="21489-104">Canonical functions cannot be extended by a provider.</span></span>  
  
 <span data-ttu-id="21489-105">Estas funciones canónicas se convertirán en la funcionalidad de origen de datos correspondiente para el proveedor.</span><span class="sxs-lookup"><span data-stu-id="21489-105">These canonical functions will be translated to the corresponding data source functionality for the provider.</span></span> <span data-ttu-id="21489-106">Esto permite que las llamadas a funciones se expresen de forma común en los orígenes de datos.</span><span class="sxs-lookup"><span data-stu-id="21489-106">This allows for function invocations expressed in a common form across data sources.</span></span>  
  
 <span data-ttu-id="21489-107">Dado que estas funciones canónicas son independientes de los orígenes de datos, los tipos de argumentos y valores devueltos de las funciones canónicas se definen en función de los tipos en el modelo conceptual.</span><span class="sxs-lookup"><span data-stu-id="21489-107">Because these canonical functions are independent of data sources, argument and return types of canonical functions are defined in terms of types in the conceptual model.</span></span> <span data-ttu-id="21489-108">Sin embargo, puede que algunos orígenes de datos no admitan todos los tipos en el modelo conceptual.</span><span class="sxs-lookup"><span data-stu-id="21489-108">However, some data sources might not support all types in the conceptual model.</span></span>  
  
 <span data-ttu-id="21489-109">Cuando las funciones canónicas se usan en una consulta de [!INCLUDE[esql](../../../../../../includes/esql-md.md)], se llamará a la función apropiada en el origen de datos.</span><span class="sxs-lookup"><span data-stu-id="21489-109">When canonical functions are used in an [!INCLUDE[esql](../../../../../../includes/esql-md.md)] query, the appropriate function will be called at the data source.</span></span>  
  
 <span data-ttu-id="21489-110">Todas las funciones canónicas tienen comportamiento de entrada NULL y condiciones de error especificadas explícitamente.</span><span class="sxs-lookup"><span data-stu-id="21489-110">All canonical functions have both null-input behavior and error conditions explicitly specified.</span></span> <span data-ttu-id="21489-111">Los proveedores de almacenamiento deben satisfacer ese comportamiento, pero [!INCLUDE[adonet_ef](../../../../../../includes/adonet-ef-md.md)] no lo exige.</span><span class="sxs-lookup"><span data-stu-id="21489-111">Store providers should comply with that behavior, but [!INCLUDE[adonet_ef](../../../../../../includes/adonet-ef-md.md)] does not enforce this behavior.</span></span>  
  
 <span data-ttu-id="21489-112">Para los escenarios LINQ, las consultas con el [!INCLUDE[adonet_ef](../../../../../../includes/adonet-ef-md.md)] implican la asignación de métodos CLR a métodos en el origen de datos subyacente.</span><span class="sxs-lookup"><span data-stu-id="21489-112">For LINQ scenarios, queries against the [!INCLUDE[adonet_ef](../../../../../../includes/adonet-ef-md.md)] involve mapping CLR methods to methods in the underlying data source.</span></span> <span data-ttu-id="21489-113">Los métodos de CLR se asignan a funciones canónicas de modo que un conjunto específico de métodos se asignará correctamente, con independencia del origen de datos.</span><span class="sxs-lookup"><span data-stu-id="21489-113">The CLR methods map to canonical functions, so that a specific set of methods will correctly map, regardless of the data source.</span></span>  
  
## <a name="canonical-functions-namespace"></a><span data-ttu-id="21489-114">Espacio de nombres de funciones canónicas</span><span class="sxs-lookup"><span data-stu-id="21489-114">Canonical Functions Namespace</span></span>  
 <span data-ttu-id="21489-115">El espacio de nombres para una función canónica es <xref:System.Data.Metadata.Edm>.</span><span class="sxs-lookup"><span data-stu-id="21489-115">The namespace for canonical function is <xref:System.Data.Metadata.Edm>.</span></span> <span data-ttu-id="21489-116">El espacio de nombres <xref:System.Data.Metadata.Edm> se incluye automáticamente en todas las consultas.</span><span class="sxs-lookup"><span data-stu-id="21489-116">The <xref:System.Data.Metadata.Edm> namespace is automatically included in all queries.</span></span> <span data-ttu-id="21489-117">Sin embargo, si se importa otro espacio de nombres que contiene una función con el mismo nombre que una función canónica (en el espacio de nombres <xref:System.Data.Metadata.Edm>), se debe especificar el espacio de nombres.</span><span class="sxs-lookup"><span data-stu-id="21489-117">However, if another namespace is imported that contains a function with the same name as a canonical function (in the <xref:System.Data.Metadata.Edm> namespace), you must specify the namespace.</span></span>
