## <a name="how-snapshot-isolation-and-row-versioning-work"></a><span data-ttu-id="a612f-156">Cómo funcionan el aislamiento de instantáneas y la versión de fila</span><span class="sxs-lookup"><span data-stu-id="a612f-156">How Snapshot Isolation and Row Versioning Work</span></span>  
 <span data-ttu-id="a612f-157">Cuando se habilita el nivel de aislamiento SNAPSHOT, cada vez que se actualiza una fila, el motor de base de datos de SQL Server almacena una copia de la fila original en **tempdb**y agrega un número de secuencia de transacción a la fila.</span><span class="sxs-lookup"><span data-stu-id="a612f-157">When the SNAPSHOT isolation level is enabled, each time a row is updated, the SQL Server Database Engine stores a copy of the original row in **tempdb**, and adds a transaction sequence number to the row.</span></span> <span data-ttu-id="a612f-158">A continuación se muestra la secuencia de eventos que tienen lugar:</span><span class="sxs-lookup"><span data-stu-id="a612f-158">The following is the sequence of events that occurs:</span></span>  
  
-   <span data-ttu-id="a612f-159">Se inicia una nueva transacción y se le asigna un número de secuencia de transacción.</span><span class="sxs-lookup"><span data-stu-id="a612f-159">A new transaction is initiated, and it is assigned a transaction sequence number.</span></span>  
  
-   <span data-ttu-id="a612f-160">El motor de base de datos lee una fila dentro de la transacción y recupera la versión de fila de **tempdb** cuyo número de secuencia es más cercana a y menor que el número de secuencia de transacción.</span><span class="sxs-lookup"><span data-stu-id="a612f-160">The Database Engine reads a row within the transaction and retrieves the row version from **tempdb** whose sequence number is closest to, and lower than, the transaction sequence number.</span></span>  
  
-   <span data-ttu-id="a612f-161">Luego, comprueba que el número de la secuencia de transacción no esté en la lista de números de secuencia de transacción de las transacciones no confirmadas activas cuando se inició la transacción de instantánea.</span><span class="sxs-lookup"><span data-stu-id="a612f-161">The Database Engine checks to see if the transaction sequence number is not in the list of transaction sequence numbers of the uncommitted transactions active when the snapshot transaction started.</span></span>  
  
-   <span data-ttu-id="a612f-162">La transacción lee la versión de la fila de **tempdb** que era actual correspondiente al inicio de la transacción.</span><span class="sxs-lookup"><span data-stu-id="a612f-162">The transaction reads the version of the row from **tempdb** that was current as of the start of the transaction.</span></span> <span data-ttu-id="a612f-163">No verá las nuevas filas insertadas después, dado que esos valores de número de secuencia serán superiores al valor del número de la secuencia de transacción.</span><span class="sxs-lookup"><span data-stu-id="a612f-163">It will not see new rows inserted after the transaction was started because those sequence number values will be higher than the value of the transaction sequence number.</span></span>  
  
-   <span data-ttu-id="a612f-164">La transacción actual verá las filas que se eliminaron después de que se inició la transacción, porque habrá una versión de fila en **tempdb** con un valor de número de secuencia inferior.</span><span class="sxs-lookup"><span data-stu-id="a612f-164">The current transaction will see rows that were deleted after the transaction began, because there will be a row version in **tempdb** with a lower sequence number value.</span></span>
