# <a name="standard-numeric-format-strings"></a><span data-ttu-id="45604-102">Cadenas con formato numérico estándar</span><span class="sxs-lookup"><span data-stu-id="45604-102">Standard Numeric Format Strings</span></span>

<span data-ttu-id="45604-103">Las cadenas de formato numérico estándar se utilizan para dar formato a tipos numéricos comunes.</span><span class="sxs-lookup"><span data-stu-id="45604-103">Standard numeric format strings are used to format common numeric types.</span></span> <span data-ttu-id="45604-104">La forma de una cadena de formato numérico estándar es `Axx`, donde:</span><span class="sxs-lookup"><span data-stu-id="45604-104">A standard numeric format string takes the form `Axx`, where:</span></span>  
  
-   <span data-ttu-id="45604-105">`A` es un carácter alfabético único denominado *especificador de formato*.</span><span class="sxs-lookup"><span data-stu-id="45604-105">`A` is a single alphabetic character called the *format specifier*.</span></span> <span data-ttu-id="45604-106">Cualquier cadena de formato numérico que contenga más de un carácter alfabético, incluido el espacio en blanco, se interpreta como una cadena de formato numérico personalizado.</span><span class="sxs-lookup"><span data-stu-id="45604-106">Any numeric format string that contains more than one alphabetic character, including white space, is interpreted as a custom numeric format string.</span></span> <span data-ttu-id="45604-107">Para obtener más información, consulte [Cadenas con formato numérico personalizado](../../../docs/standard/base-types/custom-numeric-format-strings.md).</span><span class="sxs-lookup"><span data-stu-id="45604-107">For more information, see [Custom Numeric Format Strings](../../../docs/standard/base-types/custom-numeric-format-strings.md).</span></span>  
  
-   <span data-ttu-id="45604-108">`xx` es un entero opcional denominado *especificador de precisión*.</span><span class="sxs-lookup"><span data-stu-id="45604-108">`xx` is an optional integer called the *precision specifier*.</span></span> <span data-ttu-id="45604-109">El especificador de precisión está comprendido entre el 0 y el 99 y afecta al número de dígitos del resultado.</span><span class="sxs-lookup"><span data-stu-id="45604-109">The precision specifier ranges from 0 to 99 and affects the number of digits in the result.</span></span> <span data-ttu-id="45604-110">Observe que el especificador de precisión controla el número de dígitos en la representación de cadena de un número.</span><span class="sxs-lookup"><span data-stu-id="45604-110">Note that the precision specifier controls the number of digits in the string representation of a number.</span></span> <span data-ttu-id="45604-111">No redondea el número en sí.</span><span class="sxs-lookup"><span data-stu-id="45604-111">It does not round the number itself.</span></span> <span data-ttu-id="45604-112">Para realizar una operación de redondeo, use el método <xref:System.Math.Ceiling%2A?displayProperty=nameWithType>, <xref:System.Math.Floor%2A?displayProperty=nameWithType> o <xref:System.Math.Round%2A?displayProperty=nameWithType>.</span><span class="sxs-lookup"><span data-stu-id="45604-112">To perform a rounding operation, use the <xref:System.Math.Ceiling%2A?displayProperty=nameWithType>, <xref:System.Math.Floor%2A?displayProperty=nameWithType>, or <xref:System.Math.Round%2A?displayProperty=nameWithType> method.</span></span>  
  
    <span data-ttu-id="45604-113">Cuando el *especificador de precisión* controla el número de dígitos fraccionarios de la cadena de resultado, esta refleja un número redondeado al resultado representable más cercano al resultado de precisión infinita.</span><span class="sxs-lookup"><span data-stu-id="45604-113">When *precision specifier* controls the number of fractional digits in the result string, the result string reflects a number that is rounded to a representable result nearest to the infinitely precise result.</span></span> <span data-ttu-id="45604-114">En el caso de que haya dos resultados representables igualmente cercanos:</span><span class="sxs-lookup"><span data-stu-id="45604-114">If there are two equally near representable results:</span></span>
    - <span data-ttu-id="45604-115">**En .NET Framework y .NET Core (hasta la versión 2.0)**, el runtime selecciona el resultado con el dígito menos significativo más elevado (es decir, usando <xref:System.MidpointRounding.AwayFromZero?displayProperty=nameWithType>).</span><span class="sxs-lookup"><span data-stu-id="45604-115">**On the .NET Framework and .NET Core up to .NET Core 2.0**, the runtime selects the result with the greater least significant digit (that is, using <xref:System.MidpointRounding.AwayFromZero?displayProperty=nameWithType>).</span></span>
    - <span data-ttu-id="45604-116">**En .NET Core 2.1 y versiones posteriores**, el runtime selecciona el resultado con un dígito menos significativo par (es decir, usando <xref:System.MidpointRounding.ToEven?displayProperty=nameWithType>).</span><span class="sxs-lookup"><span data-stu-id="45604-116">**On .NET Core 2.1 and later**, the runtime selects the result with an even least significant digit (that is, using <xref:System.MidpointRounding.ToEven?displayProperty=nameWithType>).</span></span>
